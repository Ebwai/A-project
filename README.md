# A-project（第三阶段：行为同步）


## 🎯 项目愿景

按照[CyberMe](https://github.com/Ebwai/CyberMe)总架构的设想，A-project 主要想解决如何让AI具有我们的经验去做事
当前公开的0.0版本 A-project，更多是为了：
- 让更多人可以通过这2个部署起来简单的工作流解放时间，去做别的事情，无需再每日花大量的时间阅读处理邮件，在有想法或想分享的东西时，也可以一键将粗糙的原始文本自动分享到社交平台（小红书，微信公众号，Bilibili，抖音等等）
- 提供一种将人的特殊性（审美，价值观，行为方式）通过提炼为工作流融入AI的一种尝试【以前的方案大多是通过给定提示词，或者用claude skill等类似的方式去赋予AI人的特殊性】
- 为后续接入「记忆系统」提供一个清晰的项目外壳【解耦插件】


🚧更完整的设想还在实践ing，合适的时间会更新出来

## 📁 当前仓库包含内容

本仓库目前主要包含2个经验内化为AI行动的工作流：

1. **读邮箱**  
   - 持续监听教育邮箱 / 工作邮箱的新邮件
   - 自动过滤垃圾信息，抓住真正重要的通知、活动、机会
   - 结合邮件正文 + 附件图片（如活动海报），做语义级理解

2. **信息分享**
   **将一份原始「信息」按照不同平台的特色风格化**       -->      **多社交平台协同分发**
   
   「**风格化**」
      - 从原始邮件中抽取关键信息：时间、地点、主题、截止日期、主办方等
      - 根据你的个人偏好，用不同风格做二次创作：
        - 适合 Bilibili / 抖音 的脚本结构与标题
        - 适合 小红书 的生活化分享文案
        - 适合 公众号 的长文结构化摘要

   「**多平台信息分发**」
      - 接入：利用B站、小红书、抖音、微信公众号 的半自动/自动发布接口（视平台开放程度而定）

## 🧠 在整体系统中的位置

- **A-project（行为层）**：
  - 更偏「具体动作」和「自动化流程」，比如：
    - 轮询邮箱
    - 解析内容
    - 生成日报
    - 推送提醒
- **M-project（记忆与心智层）**：
  - 更关注「长期偏好」和「个人记忆」，例如：
    - 你长期关心哪些主题（求职 / 海外项目 / 比赛 / 奖学金等）
    - 你想要在各平台上呈现的个人风格（理性 / 松弛 / 幽默 / 技术流）
  - 未来会逐步把这些偏好融入到每一次邮件重写和内容生成中。
